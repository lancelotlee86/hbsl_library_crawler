## 整体思路
1. 遍历所有可能的学号，获取实际存在的学号列表和邮箱；
2. 为所有的学号在mongodb中初始化一条新的纪录；
3. 开始为所有的学号每天遍历一次，并填充到mongodb中去。

### 分布式设计
设置一个主机master，对外暴露 个接口，各个接口的功能如下
1. get_next_student_account() 返回一个可用的没有被更新到数据库中的账户，如果没有下一个账户，则调用下面的接口
2. post_records_to_master() 当从机slave完成了当前的工作后，接收从机的今天的更新数据并合并到主机的数据库中。

### 主机完成的工作
1. 遍历学生账号，获取可用的账号，并存储在student_account表中。
2. 通过api给从机分配从机需要检索的账号
3. 接收从机完成任务后返回回来的数据，并合并到数据库中。
